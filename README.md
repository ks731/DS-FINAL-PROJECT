Occupational AI Exposure: Predicting Job Displacement Risk from O*NET Skill Profiles
Kevin Sanchez | Intro to Data Science | Hunter College CUNY | Spring 2026

This project builds predictive models estimating AI-driven job displacement risk for 617 unique U.S. occupations, using measurable skill, ability, and knowledge features from the O*NET database as predictors. The project identifies which specific occupational characteristics most strongly predict automation risk and visualizes how that risk is distributed across job types and U.S. states.

Data Sources
All data is publicly available.

Dataset 1 — Frey & Osborne + BLS

URL: https://www.kaggle.com/datasets/andrewmvd/occupation-salary-and-likelihood-of-automation
File used: automation_data_by_state.csv
Contains: SOC codes, occupation names, Frey & Osborne automation probability scores (0–1), and employment counts for all 51 U.S. states

Dataset 2 — O*NET Database v29.0

URL: https://www.kaggle.com/datasets/emarkhauser/onet-29-0-database
Files used: Skills.txt, Abilities.txt, Knowledge.txt
Contains: Numerical skill, ability, and knowledge domain ratings for 900+ U.S. occupations

How to Run
Run the scripts in order. Each script (except rows.py) loads merged.csv, so rows.py must be run first.

Make sure all data files and scripts are in the same directory before running.

Step 1 — Load, merge, and explore the data:

python3 rows.py

Outputs: merged.csv, eda_target_distribution.png

Step 2 — Train Ridge and Lasso regression models:

python3 model.py

Outputs: feature_importance.png, predicted_vs_actual.png

Step 3 — Run PCA and K-Means clustering:

python3 clustering.py

Outputs: elbow_plot.png, cluster_scatter.png

Step 4 — Train logistic regression classifier:

python3 classifier.py

Outputs: confusion_matrix.png, classifier_features.png

Step 5 — Generate U.S. choropleth map:

python3 visualizations.py

Outputs: us_risk_map.html (interactive), us_risk_map.png (static)


What Each Script Does
rows.py
Loads the four source data files, pivots the O*NET files from long to wide format (one row per occupation, one column per skill), merges everything on the shared SOC occupational code, and runs exploratory data analysis. Saves the final merged dataset as merged.csv for all subsequent scripts.

707 rows × 175 columns before deduplication
120 feature columns — O*NET skill, ability, and knowledge ratings (all complete, no nulls)
Target variable mean: 0.506, median: 0.600, std: 0.377
Distribution is U-shaped — spike of safe jobs near 0, rise of high-risk jobs near 1
model.py
Trains Ridge and Lasso regression models on the 120 O*NET feature columns to predict the continuous automation probability score (0–1). Uses 80/20 train/test split and cross-validation to select the best regularization strength.

Key Findings
O*NET skill profiles explain 65.7% of variance in automation probability (Ridge R²=0.657)
Jobs can be classified as high or low risk with 83.1% accuracy using skill ratings alone
Four distinct occupation clusters emerge naturally from skill profiles, ranging from avg risk 0.18 (leadership roles) to 0.82 (routine physical roles)
Nevada faces the highest workforce automation exposure; Washington D.C. the lowest
Skills most protective against automation: Deductive Reasoning, Originality, Social Perceptiveness
Skills most associated with high risk: Near Vision, Persuasion, Speed of Limb Movement


Limitations
Automation probability scores are sourced from Frey & Osborne (2013) — now over a decade old
O*NET skill ratings are self-reported and may not perfectly capture actual task composition
Some creative roles (e.g. Craft Artists, Dancers) appear in the high-risk cluster due to low cognitive complexity ratings, which may not reflect their true automation risk
The model predicts risk based on occupational skill profiles, not individual worker characteristics
